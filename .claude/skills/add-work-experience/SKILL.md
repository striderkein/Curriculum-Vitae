# add-work-experience

職務経歴書（docs/README.md）に職務経歴を対話的に追記するスキル。

## トリガー

ユーザーが「職務経歴を追加」「経歴を追記」「add work experience」等と言った場合。

## 手順

1. まず `docs/README.md` を読み、現在の職務経歴の構成を把握する。
2. ユーザーに1件ずつプロジェクト情報をヒアリングする。以下の項目を質問で引き出す：
   - プロジェクト概要（何を開発したか）
   - 期間
   - 技術スタック（言語・フレームワーク・インフラ等）
   - チーム規模
   - 役割（詳細設計、実装、テスト、コードレビュー等）
   - 担当内容の詳細
3. ユーザーが「これで全部」「以上」等と言うまで、次のプロジェクトについて質問を続ける。
4. 全件聞き終えたら、既存の職務経歴書のフォーマットに合わせて `docs/README.md` を編集する。

## ヒアリングのルール

- 1件ずつ聞く。一度に全件聞こうとしない。
- ユーザーが「以降同じ」と言った項目は、以降のプロジェクトでも同じ値を使う。
- 不明な点があればその場で補足質問する。ただし過度に細かく聞きすぎない。
- 顧客名など機密性のある情報は、ユーザーの指示に従い適度に曖昧にする（例：「大手インフラ企業」）。

## 出力フォーマット

既存の `docs/README.md` の職務経歴セクションのフォーマットに厳密に従う。典型的な構造：

```markdown
### 会社名（YYYY/MM - YYYY/MM）

概要文（1〜2文）

- プロジェクト名（期間）
  - **プロジェクト規模：**
    - N 名
  - **役割：**
    - 詳細設計、実装 等
  - **プロジェクト詳細：**
    - 箇条書きで担当内容
```
